2. HTML Document Structure

Syllabus:

  1. HTML Document Structure (Head, Body)

    • Understanding the Basic Structure:

      • Every HTML document has a basic structure that includes the <head> and <body> sections, wrapped inside the <html> tag.
      • The <head> section contains meta-information about the document, while the <body> section contains the content that is displayed on the webpage.

    • Tags Covered:

      • <html>, <head>, <body>, <title>

    • Example:

      html

      <!DOCTYPE html>
    <html>
    <head>
        <title>My Webpage</title>
    </head>
    <body>
        <h1>Welcome to My Webpage</h1>
        <p>This is an example of a basic HTML structure.</p>
    </body>
    </html>

  2. Meta Tags

    • What Are Meta Tags?
      • Meta tags provide metadata about the HTML document, such as character encoding, description, keywords, and author.
    • Common Meta Tags:
      • <meta charset="UTF-8">: Specifies the character encoding for the document, ensuring proper display of characters.
      • <meta name="description" content="This is a sample webpage.">: Provides a brief description of the webpage, often used by search engines.
      • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Controls how the webpage is displayed on different devices, crucial for responsive design.
    • Example:
      html

      <head>
        <meta charset="UTF-8">
        <meta name="description" content="Learn the basics of HTML.">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>HTML Document Structure</title>
    </head>

  3. Linking External Resources

    • Linking CSS Files:

      • The <link> tag is used to link external CSS files to style the HTML document.
      • Attributes:
        • rel="stylesheet": Specifies the relationship between the document and the linked file.
        • href="styles.css": Specifies the path to the CSS file.
      • Example:
        html

        <head>
        <link rel="stylesheet" href="styles.css">
    </head>

    • Linking JavaScript Files:

      • The <script> tag is used to link external JavaScript files to add interactivity to the webpage.
      • Attributes:
        • src="script.js": Specifies the path to the JavaScript file.
        • defer: Ensures the script is executed after the HTML document is fully parsed.
      • Example:
        html

        <body>
        <script src="script.js" defer></script>
    </body>

  4. Basic HTML Page Layout

    • Using Semantic Elements for Layout:
      • Semantic elements help structure the content of a webpage, making it more readable and accessible.
      • Key Semantic Tags:
        • <header>: Represents the introductory content or a group of navigational links.
        • <footer>: Represents the footer of the document or a section, often containing copyright information or links.
        • <nav>: Represents a section of the page that links to other pages or parts within the page.
        • <section>: Represents a standalone section, typically with a heading.
        • <article>: Represents a self-contained piece of content, such as a blog post or news article.
    • Example:
      html

      <body>
        <header>
            <h1>My Website</h1>
            <nav>
                <ul>
                    <li><a href="#home">Home</a></li>
                    <li><a href="#about">About</a></li>
                    <li><a href="#contact">Contact</a></li>
                </ul>
            </nav>
        </header>
        
        <section id="home">
            <h2>Welcome</h2>
            <p>This is the home section.</p>
        </section>
        
        <section id="about">
            <h2>About Us</h2>
            <p>Learn more about us in this section.</p>
        </section>
        
        <footer>
            <p>&copy; 2024 My Website</p>
        </footer>
    </body>

3. HTML Links and Images

Syllabus:

  1. Creating Links (<a> Tag)

    • Purpose of Links:

      • Links, or hyperlinks, are essential elements that allow users to navigate between different pages on the web or jump to sections within a page.
      • The <a> tag is used to create these links, which can point to various destinations like websites, files, or email addresses.

    • Absolute vs. Relative URLs:

      • Absolute URLs: These include the full web address (e.g., https://www.example.com/page) and are used when linking to external websites.
      • Relative URLs: These are paths relative to the current document (e.g., about.html, images/photo.jpg). They are used for linking within the same website.
      • Example:

        <!-- Absolute URL -->
    <a href="https://www.example.com">Visit Example</a>
    
    <!-- Relative URL -->
    <a href="about.html">About Us</a>

    • Target Attribute:

      • The target attribute defines how the link will be opened.
        • target="_blank": Opens the link in a new tab or window.
        • target="_self": Opens the link in the same tab (default behavior).
      • Example:

        <a href="https://www.example.com" target="_blank">Open Example in a New Tab</a>

  2. Image Embedding (<img> Tag)

    • Embedding Images:
      • The <img> tag is used to embed images in an HTML document. It is an empty tag, meaning it does not have a closing tag and is self-contained.
      • Attributes:
        • src: Specifies the path to the image file, which can be an absolute or relative URL.
        • alt: Provides alternative text for the image, useful for accessibility and when the image fails to load.
        • width and height: Control the display size of the image. Values can be in pixels (e.g., width="300") or percentage (e.g., width="50%").

      • Example:

        <img src="images/logo.png" alt="Company Logo" width="200" height="100">

  3. Image Formats (JPEG, PNG, GIF, SVG)

    • Understanding Image Formats:
      • Different image formats serve different purposes based on quality, size, and usage requirements:
        • JPEG (Joint Photographic Experts Group): Best for photographs or images with many colors. Offers good quality with compression.
        • PNG (Portable Network Graphics): Supports transparency and is ideal for images with text, logos, or graphics where quality is paramount.
        • GIF (Graphics Interchange Format): Suitable for simple graphics and animations. Limited to 256 colors, making it less ideal for photos.
        • SVG (Scalable Vector Graphics): Used for vector images that scale without losing quality, perfect for logos and icons.
      • Examples of Usage:

        <!-- JPEG for a photo -->
    <img src="images/photo.jpg" alt="Nature Photo">
    
    <!-- PNG for a logo with transparency -->
    <img src="images/logo.png" alt="Transparent Logo">
    
    <!-- GIF for an animated image -->
    <img src="images/animation.gif" alt="Loading Animation">
    
    <!-- SVG for a scalable icon -->
    <img src="images/icon.svg" alt="Scalable Icon">

4. HTML Lists and Tables

Ordered and Unordered Lists

  • Unordered Lists (<ul>):

    • Used to create a list of items where the order doesn’t matter.
    • Each item in the list is defined using the <li> tag.
    • Default bullet points are used for each list item.
    • Example:
      html

      <ul>
        <li>Item 1</li>
        <li>Item 2</li>
        <li>Item 3</li>
      </ul>

  • Ordered Lists (<ol>):

    • Used to create a list of items where the order does matter.
    • Each item is defined using the <li> tag.
    • Default numbering is used for each list item.
    • Example:
      html

      <ol>
        <li>First item</li>
        <li>Second item</li>
        <li>Third item</li>
      </ol>

Nesting Lists

  • Lists can be nested inside other lists to create a hierarchical structure.
  • You can nest an unordered list inside an ordered list, or vice versa.
  • Example:
    html

    <ul>
        <li>Item 1
          <ul>
            <li>Sub-item 1</li>
            <li>Sub-item 2</li>
          </ul>
        </li>
        <li>Item 2</li>
      </ul>

Creating Tables

Tables are used to organize data in rows and columns.

  • Table Structure:

    • <table>: The container element for all table content.

    • <tr>: Table row; used to define a row within the table.

    • <td>: Table data cell; used to define the cells within a row.

    • <th>: Table header cell; used for header cells that often appear at the top of a column.

    • <thead>: Container for the table header; used to group the header content.

    • <tbody>: Container for the table body; used to group the body content.

    • <tfoot>: Container for the table footer; used to group footer content.

    • Example:

      html

      <table>
        <thead>
          <tr>
            <th>Header 1</th>
            <th>Header 2</th>
          </tr>
        </thead>
        <tbody>
          <tr>
            <td>Data 1</td>
            <td>Data 2</td>
          </tr>
          <tr>
            <td>Data 3</td>
            <td>Data 4</td>
          </tr>
        </tbody>
        <tfoot>
          <tr>
            <td>Footer 1</td>
            <td>Footer 2</td>
          </tr>
        </tfoot>
      </table>
Table Attributes

  • border Attribute:

    • Specifies whether the table should have borders.
    • Example: <table border="1"> adds borders around the table cells.

  • cellpadding Attribute:

    • Specifies the space between the cell content and the cell borders.
    • Example: <table cellpadding="10"> increases the padding inside each cell.

  • cellspacing Attribute:

    • Specifies the space between the cells in the table.
    • Example: <table cellspacing="5"> adds space between each cell.

  • colspan Attribute:

    • Allows a cell to span across multiple columns.
    • Example: <td colspan="2"> makes the cell span across two columns.

  • rowspan Attribute:

    • Allows a cell to span across multiple rows.
    • Example: <td rowspan="2"> makes the cell span across two rows.

Node.js for Back-end Development

Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine. It allows developers to build scalable network applications and handle server-side operations.

  const http = require('http');
  const server = http.createServer((req, res) => {
      res.statusCode = 200;
      res.setHeader('Content-Type', 'text/plain');
      res.end('Hello, Node.js!\n');
  });
  server.listen(3000, () => {
      console.log('Server running at http://localhost:3000/');
  });

Building Full-stack Applications

Building full-stack applications involves integrating both front-end and back-end technologies. This includes using a front-end framework like React, and a back-end environment like Node.js, along with a database.

  const express = require('express');
  const app = express();
  app.get('/', (req, res) => {
      res.send('Hello, Full-stack Application!');
  });
  app.listen(3000, () => {
      console.log('Server running on port 3000');
  });
